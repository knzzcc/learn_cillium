## 考試當天策略（兩輪法）

**第一輪**：快速過所有題目
- 簡單題（RBAC、Pod、Service）直接做完
- 複雜題（troubleshooting、etcd）先標記跳過
- 目標：30 分鐘內完成 60% 題目

**第二輪**：處理標記題目
- 剩餘時間集中攻難題
- 不確定的題目，先寫出部分答案再說

## 常見失分陷阱

- **忘記切換 context**：每道題第一件事 `k config use-context`
- **namespace 搞錯**：題目通常會指定，仔細讀
- `k set image` 的 container 名稱要看 pod spec，不是 deployment 名稱
- etcd cert flag 三個缺一不可：`--cacert`、`--cert`、`--key`
- StorageClass 名稱大小寫要完全一致
- NetworkPolicy 預設是 deny all，記得開 DNS egress（port 53）

```yaml

- name: logging-agent
  image: fluentd:v1.17
  restartPolicy: Always   # 這行讓它變成 sidecar
```

## Gateway API 基本結構

```yaml
# Gateway
apiVersion: gateway.networking.k8s.io/v1
kind: Gateway
metadata:
  name: my-gateway
spec:
  gatewayClassName: nginx
  listeners:
  - name: http
    port: 80
    protocol: HTTP
---
# HTTPRoute
apiVersion: gateway.networking.k8s.io/v1
kind: HTTPRoute
metadata:
  name: my-route
spec:
  parentRefs:
  - name: my-gateway
  rules:
  - backendRefs:
    - name: my-service
      port: 80
```
