# k3s 安裝與基本使用

## k3s 是什麼

k3s 是 Rancher 做的輕量 Kubernetes，單一 binary（< 100MB），
設計給 edge、IoT、CI、本機開發用。功能完整但拿掉了一些 k8s 的大型元件。

---

## 安裝

### 單節點（最快，1 分鐘內）

```bash
curl -sfL https://get.k3s.io | sh -
```

裝完自動啟動，確認：

```bash
sudo k3s kubectl get nodes
# 或設定 kubeconfig 後直接用 kubectl
```

### 設定 kubeconfig（讓你可以直接用 kubectl）

```bash
mkdir -p ~/.kube
sudo cp /etc/rancher/k3s/k3s.yaml ~/.kube/config
sudo chown $USER:$USER ~/.kube/config
export KUBECONFIG=~/.kube/config

# 驗證
kubectl get nodes
```

### 多節點（1 control plane + N worker）

```bash
# control plane（記下 token 和 IP）
curl -sfL https://get.k3s.io | sh -

# 取得 token
sudo cat /var/lib/rancher/k3s/server/node-token

# worker node（換成你的 IP 和 token）
curl -sfL https://get.k3s.io | K3S_URL=https://<control-plane-ip>:6443 \
  K3S_TOKEN=<token> sh -
```

### 常用安裝選項

```bash
# 關閉內建 traefik（要裝 Istio 或自己的 ingress 時用）
curl -sfL https://get.k3s.io | INSTALL_K3S_EXEC="--disable traefik" sh -

# 關閉 traefik + servicelb（負載均衡器）
curl -sfL https://get.k3s.io | INSTALL_K3S_EXEC="--disable traefik --disable servicelb" sh -

# 指定版本（對應考試 k8s 版本）
curl -sfL https://get.k3s.io | INSTALL_K3S_VERSION="v1.35.0+k3s1" sh -

# 指定 Pod CIDR
curl -sfL https://get.k3s.io | INSTALL_K3S_EXEC="--cluster-cidr=10.42.0.0/16" sh -
```

## 基本管理指令

```bash
# 服務管理
sudo systemctl status k3s
sudo systemctl restart k3s
sudo systemctl stop k3s

# 查看 log
sudo journalctl -u k3s -f

# 節點狀態
kubectl get nodes -o wide

# 所有 namespace 的 pod
kubectl get pods -A
```

---

## 解除安裝

```bash
# control plane
/usr/local/bin/k3s-uninstall.sh

# worker
/usr/local/bin/k3s-agent-uninstall.sh
```

## k3s 的預設元件

| 元件 | 說明 |
|------|------|
| **containerd** | container runtime（不是 Docker） |
| **Flannel** | 預設 CNI（VXLAN） |
| **CoreDNS** | DNS |
| **Traefik** | 預設 Ingress controller |
| **ServiceLB** | 簡單的 LoadBalancer 實作 |
| **local-path-provisioner** | 預設 StorageClass（hostPath 動態 PV） |
| **embedded etcd 或 SQLite** | 狀態儲存（單節點用 SQLite，HA 用 etcd） |

## 實際使用

k3s 安裝好之後，kubectl 指令跟 k8s 完全一樣：

```bash
# 部署
kubectl create deployment nginx --image=nginx
kubectl expose deployment nginx --port=80 --type=NodePort

# 查看
kubectl get all
kubectl get svc

# 存取（NodePort）
kubectl get svc nginx
# 用 NodePort 的 port 存取，例如 http://localhost:30080

# 清除
kubectl delete deployment nginx
kubectl delete svc nginx
```

## 進階：用 Helm 裝東西

k3s 內建 Helm controller，可以直接 apply HelmChart CRD：

```bash
# 或直接裝 helm
curl https://raw.githubusercontent.com/helm/helm/main/scripts/get-helm-3 | bash

# 加 repo 安裝 app
helm repo add bitnami https://charts.bitnami.com/bitnami
helm install my-nginx bitnami/nginx
```
